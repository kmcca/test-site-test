archer.do

Veracode has deprecated the archer.do call. Although this call is functional, Veracode recommends that you use the asynchronous Archer API calls to avoid lengthy wait times and timeouts when retrieving large data sets.

Resource URL​

https://analysiscenter.veracode.com/api/archer.do

Parameters​

The archer.do call has no required parameters.

NameTypeDescription
app_idIntegerApplication ID of the target application for which you wish to pull data. You can find the application ID in the main Archer feed or by using the Results API.
periodStringUse to select published applications with modifications (either published or mitigated) during the specified time period. Values are: yesterday, last week (preceding Saturday to Sunday), last month (preceding calendar month)
from_dateStringSpecify the beginning of a date range using the format MM-dd-yyyy. The range begins at 12:00AM ET on the specified date. Use this parameter with the to_date parameter.
to_dateStringSpecify the end of a date range using the format MM-dd-yyyy. The range ends at 12:00AM ET the morning after the specified date. Use this parameter with the from_date parameter.
scan_typeStringSpecify a flaw type: static, dynamic, manual

HTTPie Example​

The example uses the HTTPie command-line tool.

http --auth-type=veracode_hmac -o archerfeed.xml "https://analysiscenter.veracode.com/api/archer.do"`

HTTPie Results​

The archer.do call returns the archer XML document, which references the archerreport.xsd schema file. You can use the XSD schema file to validate the XML data. The XML report lists details about the scan of your Archer application.
